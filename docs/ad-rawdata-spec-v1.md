# 广告 Raw Data 语义草案 V1

## 1. 这份补充信息带来的核心调整

这批广告 raw data 信息说明，之前方案方向是对的，但需要补 4 个关键点：

1. 广告数据智能体必须内置更强的字段歧义处理，尤其是“点位类型”和“点位名称”。
2. 成本相关指标不能只靠单一费用字段，需要先走费用口径选择规则。
3. `备注` 和 `adslot` 不能只当普通文本字段，它们要参与筛选和分析前判断。
4. 月报需要明确区分“单月分析”和“跨月分析”两种模板。

## 2. 当前广告场景的重点字段

### 2.1 核心维度字段

- 媒体名称
- 点位类型
- 定剧类型（素材 id）
- `adcode`：最细颗粒度，广告排期 id
- 广告类型：`CPM / CPC / FREE / CPD`
- 广告页落地页 URL
- 投放时间
- 车型：`ACC / BRE / P7 / INT / AVA / VEZ / ZRV ...`

补充说明：

真实文件中的 `投放时间` 是区间字符串，不是单个日期。

### 2.2 规则判断字段

- 是否接受 DMP 曝光监测
- 是否接受 DMP 点击监测
- 备注
- `adslot`

### 2.3 结果指标字段

- `PV`
- `UV`
- `CLICK`
- 官网到达
- 有效访客
- 高意向访客
- 留资数
- 净总价
- 总成本

## 3. 广告场景中的关键语义提醒

### 3.1 点位类型不能和点位名称混淆

系统必须把这类相似字段视为高风险歧义字段。

要求：

- 如果数据里同时出现“点位类型”和“点位名称”，默认不直接猜
- 应在澄清阶段明确提示用户确认
- 月报中的“分媒体点位表现”需要说明本次到底使用的是哪一个字段

### 3.2 `adcode` 是最细分析粒度

这意味着：

- 明细级排查和异常定位，优先可以落到 `adcode`
- 上卷到媒体、点位、车型时，需要避免重复聚合错误

### 3.4 `投放时间` 需要单独解析

真实数据中，`投放时间` 的值类似：

- `2025-01-01至2025-01-31`
- `2024-09-06至2025-04-08`

因此系统需要把它拆成：

- `start_date`
- `end_date`
- `time_range_text`

不能把原字段直接当作普通日期列。

### 3.3 `备注` 和 `adslot` 是业务语义入口

这两个字段不是普通备注字段，而是业务判断字段。

它们可能包含：

- 补量
- 赠送
- 录入用途但无真实投放
- 特殊定向策略
- 特殊执行情况

因此系统应把它们纳入“分析前预处理”。

## 4. 广告数据场景的费用口径

### 4.1 费用主规则

当前口径：

- 除百度品专点位外，费用默认使用 `净总价`
- 百度品专点位费用使用 `总成本`

所以 V1 不应把“费用”简单映射成某一个固定字段，而应有一个标准化费用字段：

- `effective_cost`

建议规则：

```text
if point_type is 百度品专:
  effective_cost = 总成本
else:
  effective_cost = 净总价
```

### 4.2 成本类核心指标

主指标：

- 高意向访客成本 = `effective_cost / 高意向访客`

次指标：

- 有效访客成本 = `effective_cost / 有效访客`

### 4.3 链路类指标

可用于链路拆解分析：

- CPM
- CPC
- 点击率
- 到达率
- 有效访客率
- 高意向率

### 4.4 所有率和成本都应派生计算

分析阶段不应直接采用表里现成的率和成本值，而应基于当前分析范围重新计算。

必须重算的指标至少包括：

- CPM
- CPC
- 点击率
- 到达率
- 有效访客率
- 高意向率
- 高意向访客成本
- 有效访客成本

原因：

- 原表可能没有这些字段
- 即使有，也未必符合当前筛选条件
- 只有按当前有效样本重算，结果才和本次分析口径一致

建议公式：

- `CPM = effective_cost / PV * 1000`
- `CPC = effective_cost / CLICK`
- `点击率 = CLICK / PV`
- `到达率 = 官网到达 / CLICK`
- `有效访客率 = 有效访客 / 官网到达`
- `高意向率 = 高意向访客 / 有效访客`
- `高意向访客成本 = effective_cost / 高意向访客`
- `有效访客成本 = effective_cost / 有效访客`

## 5. 必须纳入语义配置的排除规则

### 5.1 URL 过滤是前置规则

当前已明确：

- 只分析 `https://www.ghac` 开头的 URL

这应成为广告数据智能体的默认前置过滤规则，而不是每次临时提醒。

### 5.2 DMP 监测缺失规则

如果：

- 是否接受 DMP 曝光监测 = 否
- 且费用不为 0

说明：

- 数据存在缺失
- 不应纳入 CPM 等依赖曝光的成本计算

同理：

- 是否接受 DMP 点击监测 = 否
- 且费用不为 0

则：

- 不应纳入 CPC 等依赖点击的成本计算

注意：

这里不是简单“整条记录删掉”，而是要按指标类型决定是否纳入某项计算。

### 5.3 FREE 点位规则

如果广告类型为 `FREE`：

- 费用为 0 可能是补量或赠送
- 不纳入成本计算
- 不纳入成本对比

但它仍可能对流量规模、链路表现、业务说明有价值，所以不一定从整体分析里完全删除。

### 5.4 文本语义筛选规则

在 URL 过滤之后，还要对 `备注` 和 `adslot` 做语义判断。

目的：

- 判断是否应该筛掉
- 判断是否需要单独标记
- 判断是否应在报告中解释为特殊样本

V1 建议至少支持 3 类分类结果：

- `exclude`
- `include_with_warning`
- `normal`

### 5.5 缺失数据与不可监测点位处理

广告数据里会存在不可监测点位，这类记录通常会带来链路缺失。

当前已知判断信号：

- 不接受 DMP 曝光监测
- 不接受 DMP 点击监测

这类数据大概率导致以下字段缺失或不可靠：

- `PV`
- `CLICK`
- 官网到达
- 有效访客
- 高意向访客

系统不能简单把缺失当 0 然后继续计算所有率和成本。

建议处理原则：

1. 先识别是否为不可监测样本
2. 判断当前指标是否依赖缺失字段
3. 依赖则不纳入该指标计算
4. 在结果中明确说明哪些样本未纳入哪些指标

重点：

- “是否参与分析”不是一个总开关
- 应按指标类型决定是否参与计算

补充观察：

真实样本中，`是否接受DMP曝光监测 = 否` 的记录里，虽然 `PV` 为 0，但 `CLICK`、`官网到达`、`有效访客`、`高意向访客` 仍可能有值。

因此这个规则必须坚持按指标类型生效，不能整条排除。

## 6. 数据预处理顺序建议

广告数据智能体建议采用以下处理顺序：

1. 文件读取
2. 字段识别
3. 解析 `投放时间` 区间
4. URL 前缀过滤
5. 费用字段标准化为 `effective_cost`
6. `备注` / `adslot` 文本语义分类
7. 识别不可监测样本与缺失模式
8. 应用默认排除与警告规则
9. 输出待确认项
10. 用户确认后执行正式分析

## 7. 广告场景的待确认项建议

上传后建议优先确认这些内容：

1. 本次“点位”分析使用 `点位类型` 还是 `点位名称`
2. 百度品专点位是否按 `总成本` 计算
3. `FREE` 点位是否完全排除出成本相关结论
4. `备注` / `adslot` 中识别出的特殊样本是否排除
5. 本次分析是否按车型拆解
6. 当前时间范围是否正确

## 8. 月报结构调整建议

## 8.1 单月 rawdata

建议结构：

1. 整体情况
2. 分车型表现
3. 其他洞察

其中：

- 整体情况：分车型展示高意向访客成本和有效访客成本概览
- 分车型表现：重点看分媒体 / 点位表现、链路拆解、洞察、归因、下一步行动
- 其他洞察：承接补充发现、特殊样本说明、横向异常

## 8.2 多月 rawdata

建议结构：

1. 整体情况
2. 媒体趋势
3. 分车型表现
4. 其他洞察

比单月多出的关键部分是：

- 跨月媒体趋势
- 跨月车型对比
- 具体媒体或点位的趋势变化
- 跨车型共性结论提炼

## 8.3 可视化要求

月报必须带可视化展示，至少包括：

- 指标卡
- 趋势图
- 分媒体 / 点位对比图
- 分车型对比图

## 9. 对现有方案的影响

这批 raw data 信息说明，V1 里必须把以下能力从“建议有”提升到“必须有”：

- 字段歧义确认
- 费用口径配置
- 派生指标重算
- 按指标类型生效的排除规则
- 缺失数据与不可监测处理
- 文本字段语义预处理
- 单月 / 多月双模板月报

## 10. 当前结论

广告数据智能体的关键难点已经非常明确：

不是会不会算指标，而是能不能在复杂字段、复杂口径、复杂排除条件下，先把分析边界定义清楚，再去生成可信的分析结果。
