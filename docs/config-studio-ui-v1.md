# Config Studio UI V1

## 1. 目标

本文件定义 Data Agent 的“配置层界面”形态。

核心原则：

- 配置结果必须是结构化的
- 自然语言只作为辅助输入方式
- 用户最终要确认结构化配置

## 2. 产品定位

这个界面不应该只是一个“大输入框”。

它应该是一个：

可视化配置工作台 + 自然语言助手

也就是：

- 主体是结构化配置面板
- 旁边有自然语言助手，帮助用户更快生成或修改配置

## 3. 推荐页面结构

### 左侧

- 智能体列表
- 新建智能体
- 配置版本列表

### 中间

- 当前配置模块编辑区
- 表单、表格、规则卡片

### 右侧

- 自然语言配置助手
- 解析结果预览
- 风险提示
- 建议修改

## 4. 配置层界面的 7 个核心模块

### 4.1 基础信息

配置内容：

- 智能体名称
- 场景说明
- 适用文件类型
- 常见任务类型

适合自然语言辅助：

- “这是一个用于分析广告投放数据的智能体”

最后落成结构化字段。

### 4.2 字段定义

配置内容：

- 原始字段名
- 标准字段名
- 字段说明
- 字段类型
- 是否可分组
- 是否是相似字段

这里最好是表格型界面。

自然语言助手适合做：

- 扫描上传文件后自动生成字段初稿
- 识别相似字段
- 给出字段解释建议

### 4.3 指标定义

配置内容：

- 指标名称
- 指标公式
- 依赖字段
- 聚合方式
- 缺失值策略

这里最好是公式卡片式界面。

自然语言助手适合做：

- “高意向访客成本 = 费用 / 高意向访客”
- 然后系统自动翻成结构化配置

### 4.4 规则配置

配置内容：

- 默认过滤规则
- 排除规则
- 指标级排除规则
- 条件判断规则
- 特殊样本规则

这里最好是规则卡片流。

例如每张卡片展示：

- 规则名称
- 触发条件
- 生效范围
- 动作

自然语言助手适合做：

- “FREE 点位不纳入成本计算”
- “如果备注里有补量，则标记为特殊样本”

### 4.5 澄清机制

配置内容：

- 哪些情况必须让用户确认
- 卡片标题
- 卡片文案
- 选项
- 是否必选

这里应该是卡片编辑器。

自然语言助手适合做：

- “如果点位类型和点位名称同时出现，就让用户选”

### 4.6 报告模板

配置内容：

- 模板名称
- 适用场景
- 报告章节
- 每个章节依赖的数据
- 输出组件

这里最好是章节式编辑器。

自然语言助手适合做：

- “月报分为整体情况、媒体趋势、分车型表现、其他洞察、结论建议”

### 4.7 执行流程

配置内容：

- 上传后先做什么
- 哪一步需要确认
- 哪一步生成计划
- 哪一步执行分析
- 哪一步生成报告

这里最好是流程步骤编辑器。

## 5. 自然语言助手应该怎么参与

建议它主要做 4 类事：

1. 生成初稿
2. 翻译成结构化配置
3. 检查冲突和缺漏
4. 给出修改建议

不建议它直接做最终保存。

推荐交互方式：

1. 用户输入自然语言
2. 系统生成结构化候选配置
3. 用户确认或修改
4. 再保存

## 6. 一个最典型的交互例子

用户输入：

“如果广告类型是 FREE，就不要纳入成本对比；如果备注里有补量，要标记成特殊样本。”

系统右侧解析后输出：

- 规则 1
  - 类型：指标级排除规则
  - 条件：广告类型 = FREE
  - 作用范围：成本相关指标
- 规则 2
  - 类型：文本语义规则
  - 字段：备注
  - 关键词：补量
  - 分类：include_with_warning

然后用户确认。

## 7. 为什么不能只做自然语言配置

因为纯自然语言配置会导致：

- 表达歧义
- 执行不稳定
- 难以校验
- 难以精确修改
- 难以长期维护

所以自然语言应是“生成器”和“助手”，不是最终配置本体。

## 8. 为什么不能只做表单配置

因为如果完全靠表单：

- 用户学习成本太高
- 首次配置会很慢
- 高复杂度规则不容易描述

所以只做纯表单也不够好。

## 9. 最佳形态

最佳形态就是：

结构化配置为主，自然语言辅助为辅。

也就是：

- 数据最终以结构化方式保存
- 输入过程允许自然语言
- 保存前必须转成结构化配置并确认

## 10. 对你这个产品的意义

这意味着以后你要新增其他分析智能体时：

- 不需要重新找研发讲一遍逻辑
- 可以先通过配置层去完成大部分业务适配
- 只有真正超出配置模型的需求，才需要改代码

## 11. 当前建议的实现顺序

V1 建议先做这 4 个页面：

1. 智能体基础信息页
2. 字段和指标配置页
3. 规则和澄清卡片配置页
4. 报告模板配置页

自然语言助手可以先挂在右侧，作为辅助解析器，而不是一开始就承担全部配置工作。
