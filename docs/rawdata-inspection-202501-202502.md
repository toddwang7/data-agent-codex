# Rawdata Inspection Notes

## 文件

- `rawdata202501.xlsx`
- `rawdata202502.xlsx`

## 1. 结构结论

两个文件结构一致：

- 均只有一个 sheet：`raw_data`
- 均为 63 列
- 202501 文件有 381 条数据
- 202502 文件有 376 条数据

这说明广告数据智能体可以先按“固定 schema + 月度文件输入”的方式设计。

## 2. 真实字段情况

你前面口述的重点字段，文件里基本都存在，而且还有一些额外字段：

- `点击率`
- `到达率`
- `有效访客成本`
- `高意向访客成本`
- `留资率`
- `留资成本`
- `官网到达UV`
- `有设备ID的有效访客数`

其中最重要的判断是：

这些“率”和“成本”字段虽然存在，但不能直接信任，应按分析口径重新计算。

## 3. 关键观察

### 3.1 `点位类型` 和 `点位名称` 的区别是实打实存在的

两个字段都在表里，而且值差异明显：

- `点位类型` 更像分类层级，例如 `信息流/开屏`、`百度品专`
- `点位名称` 更像具体点位，例如 `RTB找回投放`、`联合品专（PC高级样式+PC&无线千人千面）`

所以这部分必须保留澄清确认。

### 3.2 `投放时间` 不是单个日期，而是区间字符串

真实值类似：

- `2025-01-01至2025-01-31`
- `2025-02-05至2025-02-28`
- `2024-09-06至2025-04-08`

这意味着：

- 不能把 `投放时间` 直接当成标准日期列
- 需要单独解析出 `start_date` 和 `end_date`
- 某些排期是跨多个月甚至跨大半年

### 3.3 文件名月份和行内 `投放时间` 不是一回事

例如：

- `rawdata202501.xlsx` 里虽然有大量 `2025-01-01至2025-01-31`
- 但也有很多 `2024-09-06至2025-04-08` 之类的长周期数据

这说明：

- “单月文件”不能简单理解为“文件中所有行都只属于这个月”
- 月报逻辑需要明确“报告月份”来自哪里

当前更稳妥的做法是：

- 优先从文件名或用户选择得到“本次报告月份”
- 再把长周期排期作为当月样本去分析，而不是试图只靠 `投放时间` 自动推断报告月份

### 3.4 URL 过滤规则成立，但会筛掉一部分数据

统计结果：

- 202501 文件中，`https://www.ghac` 开头的 URL 有 324 条
- 202502 文件中，`https://www.ghac` 开头的 URL 有 276 条

所以这个过滤规则是必要的，而且会对分析结果产生明显影响。

### 3.5 `是否接受DMP曝光监测 = 否` 不等于整条记录都没用

真实样本里存在这种情况：

- `PV = 0`
- 但 `CLICK`、`官网到达`、`有效访客`、`高意向访客` 仍有值

例如百度品专 FREE 样本就很典型。

所以必须坚持：

- 不做“整条记录一刀切排除”
- 只对依赖曝光的指标做排除
- 其他可计算指标仍可参与

### 3.6 `是否接受DMP点击监测` 在这两个文件里全部为 `是`

当前样本里没有看到“点击监测为否”的记录。

这说明：

- 点击监测缺失规则仍要保留
- 但当前样本还不足以验证它的表现

### 3.7 `备注` 和 `adslot` 都确实重要

观察到两类典型情况：

1. `备注` 中有长文本业务说明，尤其是费用分摊和采买信息
2. `adslot` 中有大量策略信息、定向说明、补量标记，甚至还有公式样式字符串

例如：

- `补量`
- `小程序落地页不可加adcode，此adcode只为出DMP代码使用`
- `OCPX+京东车主包`
- `OCPX+召回高意向访客`

所以：

- `备注` 和 `adslot` 绝对不能只当展示字段
- 它们要进入预处理和洞察提炼

## 4. 现有方案需要新增的地方

### 4.1 新增“投放时间解析规则”

需要把 `投放时间` 从原始字符串解析成：

- `start_date`
- `end_date`
- `time_range_text`

必要时再引入：

- `reporting_month`

### 4.2 月报月份不能只靠行内日期判断

更合理的方式：

- 上传时识别文件名中的月份标签
- 或由用户确认“本次报告按哪个月份输出”

### 4.3 派生指标列一律不直接信

真实数据里：

- `点击率`
- `到达率`
- `有效访客成本`
- `高意向访客成本`
- `留资率`
- `留资成本`

这些列里同时混有：

- `/`
- `0`
- 数值

这说明这些列更像原始参考，不适合作为最终分析结果源。

## 5. 当前判断

真实文件验证后，原方案整体方向是对的，但现在可以更明确地说：

广告数据智能体的关键不是“读 Excel”，而是：

- 识别这是一个带复杂业务语义的固定 schema 文件
- 用规则解释字段和排除条件
- 明确报告月份和分析范围
- 基于当前口径重算指标
- 再输出问答和月报
