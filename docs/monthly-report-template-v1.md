# Monthly Report Template V1

## 1. 目标

本文件定义广告数据智能体的“月报生成”骨架。

这里先不追求最终文案完美，而是先把结构和生成逻辑定住。因为月报是你明确提到的高价值场景，而且它和普通问答的交互、输出、确认机制都不同。

## 2. 月报场景与普通问答的区别

普通问答更像即时分析：

- 用户问一个问题
- 系统回答一个问题
- 输出可以偏轻量

月报不是这样。月报要求：

- 有固定结构
- 有稳定逻辑
- 能承接业务汇报
- 有更明确的版式和章节
- 在生成前必须确认口径

所以月报应该被视为一个模板化分析任务，而不是一条普通消息。

## 3. 月报生成前必须确认的内容

V1 建议月报在执行前至少确认以下项目：

1. 时间范围
2. 数据范围
3. 主要指标
4. 核心拆解维度
5. 是否启用默认排除规则
6. 是否存在字段映射歧义

如果其中任一项不明确，不应直接开始生成月报。

## 4. 建议的月报主结构

基于当前广告 raw data 场景，月报应区分为两种模板：

- 单月模板
- 跨月模板

两者共享核心章节，但跨月模板需要额外增加趋势比较层。

### 4.1 报告头部

包含：

- 报告标题
- 时间范围
- 数据来源文件
- 生成时间
- 关键口径说明

### 4.2 总体表现概览

用于快速回答“这个月整体怎么样”。

建议展示：

- 核心指标卡
- 环比 / 同比变化
- 关键结论摘要

### 4.3 趋势变化分析

回答“这个月过程中发生了什么变化”。

建议展示：

- 日趋势或周趋势图
- 高低峰波动
- 关键节点说明

### 4.4 关键维度拆解

回答“表现差异主要来自哪里”。

建议至少支持：

- 渠道 / 媒体拆解
- campaign 拆解
- creative 拆解
- placement 拆解

具体展示哪些维度，应基于可用字段和用户确认结果。

### 4.5 异常与原因初判

回答“哪些地方值得重点关注，以及为什么”。

V1 先做“初判”，不做复杂因果归因。

建议包括：

- 异常指标
- 异常时间点
- 异常维度
- 初步原因解释
- 需要补充确认的地方

### 4.6 结论与建议

用于最终汇报。

建议包括：

- 本月整体结论
- 优势点
- 风险点
- 下月建议动作

## 5. 月报的最小输出形态

V1 不一定一开始就输出一个最终导出的正式文档，但右侧报告面板至少要支持以下模块化展示：

- 报告标题区
- 指标卡区
- 图表区
- 分章节文本区
- 口径说明区

## 5.1 单月模板建议

单月模板建议章节：

1. 报告头部
2. 整体情况
3. 分车型表现
4. 其他洞察
5. 结论与行动建议

其中：

- 整体情况要先展示分车型的高意向访客成本和有效访客成本概览
- 分车型表现中要支持媒体 / 点位拆解、链路拆解、洞察和行动建议
- 这里的“点位”必须明确是 `点位类型` 还是 `点位名称`

## 5.2 跨月模板建议

跨月模板建议章节：

1. 报告头部
2. 整体情况
3. 媒体趋势
4. 分车型表现
5. 其他洞察
6. 结论与行动建议

新增的关键章节是“媒体趋势”，用于承接跨月对比。

在跨月模板中，“分车型表现”也建议支持：

- 跨月对比
- 具体媒体或点位趋势
- 跨车型共性结论提炼

这样后续再导出 PDF 或其他格式时，结构也能复用。

## 6. 月报生成流程建议

### 6.1 用户发起

用户可以通过两种方式触发：

- 点击“生成月报”按钮
- 在对话中输入“生成这份数据的月报”

### 6.2 系统预检查

系统先检查：

- 是否已上传文件
- 是否识别到日期字段
- 是否识别到核心指标字段
- 是否有相似字段冲突
- 是否有必要的时间范围
- 是否为单月任务还是跨月任务

### 6.3 澄清与确认

在聊天区域输出卡片式确认项：

- 本次报告时间范围是否为 2026-03-01 至 2026-03-31
- 核心指标是否使用花费、点击、曝光、转化
- 点位分析是否优先使用点位名称
- 是否排除测试 URL
- `FREE` 点位是否排除出成本对比
- `备注` / `adslot` 识别出的特殊样本是否排除或标注

用户可以直接在气泡内点击确认。

### 6.4 生成分析计划

计划建议包含：

1. 数据检查
2. 指标计算
3. 趋势分析
4. 维度拆解
5. 异常识别
6. 报告生成

### 6.5 用户确认计划

只有确认后才执行。

### 6.6 执行与流式产出

执行时，左侧展示：

- 当前步骤
- 已完成步骤
- 若有补充问题则继续确认

右侧展示：

- 报告逐段写入
- 图表逐步出现
- 指标卡逐步刷新

## 7. 月报模板建议的章节数据依赖

### 7.1 总体表现概览

依赖：

- 日期字段
- 核心指标字段
- 对比周期规则

### 7.2 趋势变化分析

依赖：

- 日期字段
- 至少一个可聚合核心指标

### 7.3 关键维度拆解

依赖：

- 至少一个稳定维度字段
- 维度字段已完成歧义确认

### 7.4 异常与原因初判

依赖：

- 时间趋势结果
- 维度拆解结果
- 异常检测规则

## 7.5 广告月报中的关键指标建议

当前广告场景中，月报应优先围绕以下指标：

- 高意向访客成本
- 有效访客成本
- PV
- UV
- CLICK
- 官网到达
- 有效访客
- 高意向访客
- 留资数

链路拆解可按需要组合：

- CPM
- CPC
- 点击率
- 到达率
- 有效访客率
- 高意向率

## 8. 月报模板的配置化建议

不要把月报结构写死在代码里。

建议模板采用配置结构：

```json
{
  "template_id": "ad-monthly-report-v1",
  "name": "广告月报",
  "trigger_intents": ["monthly_report"],
  "required_confirmations": [
    "time_range",
    "primary_metrics",
    "dimension_priority",
    "default_filters"
  ],
  "sections": [
    {
      "section_id": "overview",
      "title": "总体表现概览",
      "required_data": ["core_metrics_summary"],
      "output_blocks": ["metric_cards", "summary_text"]
    },
    {
      "section_id": "trend",
      "title": "趋势变化分析",
      "required_data": ["time_series_metrics"],
      "output_blocks": ["trend_chart", "analysis_text"]
    }
  ]
}
```

## 9. 月报的文案风格建议

V1 文案建议坚持 3 个原则：

- 结论先行
- 口径明确
- 不装作确定

也就是说：

- 有结论就明确说
- 需要说明口径就写清楚
- 如果只是初判，就标明“初步判断”或“可能原因”

## 10. 右侧报告面板建议

右侧不要只显示一大段 Markdown 文本。

建议最小组件包括：

- 章节导航
- 指标卡
- 图表卡片
- 摘要块
- 风险提示块
- 口径说明块

这会比纯文本更像真正能汇报的报告。

## 11. V1 月报范围建议

### 必做

- 固定章节骨架
- 关键指标卡
- 趋势图
- 维度拆解
- 结论摘要
- 口径说明
- 单月 / 跨月模板区分
- 分车型可视化展示
- 分媒体 / 点位可视化展示

### 暂不强求

- 自动生成特别复杂的视觉排版
- 高级因果归因
- 自动对接外部汇报系统
- 一键生成最终正式 PPT

## 12. 当前结论

月报不是聊天结果的一个变体，而是 Data Agent 中一个独立而高价值的结构化任务类型。
